Why mark-down.js

Everything you need, nothing you don't

Built on Web Components standards. No build step, no npm, no framework required.

⚡

Zero dependencies

Pure vanilla JavaScript. One .js and one .css file. No npm, no bundler, no configuration.

🧩

Web Components

Built on the native Custom Elements API. Works in all modern browsers alongside any framework — or none at all.

🛡️

Shadow DOM isolation

Rendered inside Shadow DOM so styles never bleed in or out. Your page and your markdown stay completely separate.

✂️

Smart auto-dedent

Write naturally indented markdown inside your HTML. Common leading whitespace is stripped automatically.

🎨

Inherits your colors

The stylesheet uses currentColor and relative units, adapting to your page's text color and font size.

📐

Full markdown syntax

Headings, bold, italic, strikethrough, code blocks, blockquotes, lists, tables, links, images, and rules.

🔄

Dynamic content ready

Uses connectedCallback so elements added to the DOM after page load render automatically.

🪶

Tiny footprint

The entire parser and renderer is ~200 lines of readable JavaScript. Easy to audit, easy to fork.

Get started

Up and running in 3 steps

No configuration required. Add two files and start writing markdown in your HTML.

1

Download the files

Grab mark-down.js and mark-down.css and put them in your project. That's the entire install.

2

Add the script tag

Include mark-down.js in your <head> with defer. The CSS loads automatically from the same folder.

3

Write markdown

Place <mark-down> anywhere in your page body, write markdown inside, and open it in a browser.

<!-- 1. Load the library once in <head> -->
<script src="mark-down.js" defer></script>

<!-- 2. Use it anywhere in <body> -->
<mark-down>
  # Hello, World!

  This is **bold**, this is *italic*, and this is `inline code`.

  - One item
  - Another item
  - A third item

  > A blockquote with *formatted* content inside.

  ```js
  console.log("Hello from a fenced code block!");
  ```
</mark-down>

Supported syntax

All standard markdown constructs — write it exactly the way you already know.

MarkdownOutputNotes
# Heading 1 H1 through H6 One to six # characters followed by a space
**bold** bold Also works with __double underscores__
*italic* italic Also works with _single underscores_
***bold italic*** bold italic Triple asterisks for both at once
~~strikethrough~~ strikethrough Double tildes
`inline code` inline code Single backtick; content is HTML-escaped
```lang … ``` Code block Fenced with triple backticks; optional language hint
- item Unordered list Bullet characters: -, *, or +
1. item Ordered list Any digit followed by a period and space
> quote Blockquote Markdown renders recursively inside blockquotes
--- Horizontal rule Three or more: ---, ***, or ___
[text](url) Link Standard inline link syntax
![alt](url) Image Renders responsive with max-width: 100%
| col | col | Table GFM-style pipe table; requires header separator row
two trailing spaces Line break Hard <br> within a paragraph
